## 3) Session / cookies

- Session is managed by Supabase client (createClient from `@/lib/supabase/client` on login; server uses `updateSession` from `@/lib/supabase/middleware`).
- Cookies are set by Supabase Auth; `updateSession` refreshes and forwards session to response.
- **Recommendation:** Confirm in production that after login the next request to a protected path includes the session cookie (e.g. via /api/auth/diag or manual test).
