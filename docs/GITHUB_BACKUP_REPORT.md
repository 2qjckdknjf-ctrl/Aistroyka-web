## Опционально: авто-бэкап

Для регулярного резервного копирования можно использовать один из вариантов:

1. **Pre-push hook**  
   В `.git/hooks/pre-push` перед каждым `git push` можно проверять, что текущая ветка запушена (ничего лишнего не делаем, только напоминание).

2. **GitHub Actions**  
   Workflow по расписанию (cron), который клонирует репозиторий, при необходимости создаёт коммит с датой и пушит в отдельную ветку `backup/YYYY-MM-DD` или в `main` (осторожно с правами).

3. **Локальный cron**  
   Скрипт на сервере/ноутбуке: раз в день `git add -A && git commit -m "backup: daily" && git push` (желательно в отдельную ветку или отдельный remote).

Рекомендация: при уже настроенном CI (push в `main`/ветки) дополнительный авто-бэкап через Actions обычно не нужен; при необходимости достаточно редких ручных коммитов «backup: snapshot» и push в `main`.
