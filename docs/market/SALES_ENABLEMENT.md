# Sales Enablement

**Phase 10 — Market Expansion & Revenue Scaling**  
**Pitch, value prop, ROI, objections, demos, personas, funnel.**

---

## Product pitch (60 seconds)

“AISTROYKA is a field visibility and report platform for construction and field teams. Managers assign tasks; workers complete them and submit photo reports from the field; managers review in one place. Everything syncs in real time, with full audit trail. We add optional AI to analyze reports and spot issues. So you get one source of truth, faster review, and fewer missed or duplicate reports—without spreadsheets or WhatsApp. We offer Free, Pro, and Enterprise tiers; you can start with one team and scale.”

---

## Value proposition

- **Visibility:** Who did what, when—tasks, reports, and reviews in one dashboard.
- **Speed:** Assign and review in minutes; no chasing paper or email.
- **Audit:** Full trail for compliance and handover.
- **AI (optional):** Image analysis and summaries to catch issues and save manager time.
- **Mobile-first:** Workers use native app; offline queue and sync for tough job sites.

**Proof points:** Pilot success metrics (review turnaround, activation); case study quotes when available.

---

## ROI calculator logic

- **Inputs:** Number of managers, number of workers, hours per week spent on report chasing and review, average hourly cost.
- **Savings:** (Hours saved per week × hourly cost × 52) − subscription cost. Assume conservative % time saved (e.g. 20–30%) from centralized workflow and fewer follow-ups.
- **Optional:** Add “cost of one missed defect” or “cost of delayed handover” to quantify risk reduction.
- **Output:** Annual savings; payback period (months); 3-year TCO vs status quo.
- **Tool:** Simple spreadsheet or web calculator; no code in core product.

## Objection handling

| Objection | Response |
|-----------|----------|
| “We already use X.” | “AISTROYKA focuses on the field-report loop and visibility. We can run a short pilot alongside X and show impact on review time and audit.” |
| “Our workers won’t adopt another app.” | “One app for tasks and reports; we keep it simple. Pilots show strong adoption when workers see tasks and submit in one place.” |
| “What about offline?” | “Workers queue reports offline; they sync when back online. Built for field conditions.” |
| “We need SSO / security review.” | “Enterprise tier supports SSO and we have security documentation. We can start with email auth for pilot and add SSO for rollout.” |
| “Price seems high.” | “Free tier for small teams; Pro scales with you. Pilot is free or discounted so you measure value before committing.” |
| “We need custom fields / integrations.” | “Core is assign–report–review. Custom fields and integrations are on the roadmap; pilot feedback shapes priority.” |
