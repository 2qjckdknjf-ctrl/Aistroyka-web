# Demo and Sales Enablement Kit

**Phase 8 — Pilot Rollout & Growth**  
**Materials for 10-minute demo and sales conversations.**

---

## Demo scenario (10 min)

| Minute | Action | Message |
|--------|--------|---------|
| 0–1 | Intro | “AISTROYKA connects field workers and managers: assign tasks, submit reports with photos, review in one place.” |
| 1–3 | **Manager view (web or iOS Manager)** | Log in as manager. Show Dashboard, one project, list of tasks. “Manager assigns tasks to workers; workers see them in the app.” |
| 3–5 | **Assign task** | Assign one task to a worker. “Assignment is instant; worker gets notification.” |
| 5–7 | **Worker view (iOS Worker)** | Log in as worker (or show recording). Show Today’s tasks, open task, create report, add photo, submit. “Worker submits report with photo; it syncs even from the field.” |
| 7–9 | **Review** | Back to manager. Open Reports inbox; open submitted report; approve or request changes. “Manager reviews in one click; turnaround is visible.” |
| 9–10 | **Value + next** | “You get full visibility, fewer missed reports, and faster review. We’re running pilots now—next step is a short pilot with your team.” |

**Stick to one project and one task for simplicity.** Use demo data set (below).

---

## Demo data set

- **Tenant:** One demo tenant (staging or dedicated demo env). No production data.
- **Project:** One project, e.g. “Demo Site A”.
- **Tasks:** 3–5 tasks, e.g. “Safety check”, “Daily inspection”, “Photo log”.
- **Users:** 1 manager (demo manager), 1 worker (demo worker). Credentials in secure sheet; never in slides.
- **Reports:** 1–2 pre-submitted reports so Reports inbox is not empty at start of demo.
- **Refresh:** Reset or clone demo tenant between demos if needed so data is consistent.

---

## Key value story

- **Before:** Tasks in spreadsheets or email; reports scattered; no single place to see who did what and when; review is slow.
- **After:** One place to assign tasks; workers get tasks and submit reports with photos from the field; managers see all reports and review in one click; full audit trail and faster turnaround.
- **Outcome:** Visibility, fewer missed or duplicate reports, faster review, and (with AI) smarter insights on reports when needed.

**One-liner:** “Assign tasks, get photo reports from the field, review in one place—with full visibility and audit.”

## Before/after workflow

| Step | Before | After |
|------|--------|--------|
| Assign | Email or paper list | In-app assign; worker sees in real time. |
| Do work | Worker writes on paper or separate app | Worker uses AISTROYKA; task and report linked. |
| Submit | Photo via WhatsApp or email | Report + photo in app; syncs when online. |
| Review | Manager chases or checks folder | Reports inbox; one-click approve or request changes. |
| Audit | Search emails or files | Full trail: who, when, what. |

3. **Solution:** Assign → do → report → review in one platform; web + mobile.
4. **Demo:** (Live or video) Manager assigns → Worker submits → Manager reviews.
5. **Value:** Visibility, speed, audit trail, optional AI.
6. **Pricing:** Free / Pro / Enterprise (high-level; details in PRICING_AND_PACKAGING).
7. **Next step:** Pilot with your team (4 weeks); we support onboarding and feedback.
8. **Contact:** Name, email, calendar link.

**Keep to 10–15 slides; demo is the center.**

## Objection handling guide

| Objection | Response |
|-----------|----------|
| “We already use X for tasks.” | “AISTROYKA focuses on the field report loop and visibility. We can run a short pilot alongside X and show the difference in review time and audit.” |
| “Our workers won’t use another app.” | “One app for tasks and reports; we keep it simple. Pilots show workers adopt when they see tasks and submit in one place.” |
| “What about offline?” | “Workers can create and queue reports offline; they sync when back online. We built for field conditions.” |
| “We need SSO / security review.” | “Enterprise tier supports SSO and we can provide security documentation. For pilot we can start with email/password and scope to one team.” |
| “Price seems high.” | “Free tier for small teams; Pro scales with you. Pilot is free or discounted so you can measure value before committing.” |
| “We need custom fields / integrations.” | “Core workflow is assign–report–review. Custom fields and integrations are on the roadmap; pilot feedback shapes priority.” |

**Rule:** Acknowledge, give short answer, offer pilot or follow-up. Don’t over-promise features not yet built.
