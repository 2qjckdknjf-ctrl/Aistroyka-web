# Письмо подтверждения при регистрации не приходит

Приложение при регистрации говорит «Проверьте почту», но письмо не приходит. Так бывает из‑за настроек Supabase.

---

## Почему так

У Supabase по умолчанию включена **подтверждение email** (Confirm email). Письма отправляет встроенная система Supabase:

- она **демонстрационная**: лимит около 2 писем в час;
- письма часто попадают в **спам** или **не доходят** (почтовые сервера блокируют отправителя);
- для продакшена Supabase рекомендует настроить **свой SMTP**.

Поэтому «проверь почту» есть, а письма нет.

---

## Вариант 1: Отключить подтверждение email (быстро, для теста/MVP)

Тогда пользователь сразу после регистрации сможет войти **без перехода по ссылке из письма**.

1. Открой **https://supabase.com/dashboard** → выбери свой проект.
2. Слева: **Authentication** (Аутентификация).
3. Вкладка **Providers** (Провайдеры).
4. Нажми **Email**.
5. Найди опцию **«Confirm email»** (Подтверждение email) и **выключи** её (toggle в положение Off).
6. Нажми **Save**.

После этого новые регистрации не будут требовать подтверждения по почте — можно сразу заходить по логину и паролю. Старые пользователи, которые уже зарегистрировались, но не подтвердили почту, могут войти по тому же паролю (если подтверждение отключено, Supabase может считать их «подтверждёнными» в зависимости от версии; при необходимости таких пользователей можно подтвердить вручную в разделе Authentication → Users).

---

## Вариант 2: Настроить свою почту (Custom SMTP) для продакшена

Чтобы письма реально отправлялись и не попадали в спам:

1. В том же проекте: **Authentication** → **SMTP Settings** (или **Custom SMTP**).
2. Включи **Custom SMTP** и укажи данные своего почтового сервиса, например:
   - **Resend** (resend.com),
   - **SendGrid** (sendgrid.com),
   - **Brevo** (brevo.com),
   - **AWS SES** и т.п.
3. Укажи: host, port, user, password, From-адрес (например `noreply@aistroyka.ai`).
4. Сохрани настройки.

После этого письма подтверждения будут уходить через твой SMTP и с большей вероятностью дойдут до inbox.

---

## Что проверить ещё

- **Спам/папка «Промоакции»** — иногда письмо от Supabase уже пришло, но попало не во «Входящие».
- **Auth → Users** — пользователь с твоей почтой создан; если у него статус «Unconfirmed», после отключения Confirm email он всё равно сможет войти по паролю (или подтверди его вручную кнопкой в Dashboard).
- **Authentication → URL Configuration** — **Site URL** и **Redirect URLs** должны содержать твой домен (например `https://aistroyka.ai`), иначе ссылка из письма может вести не туда.

Итого: для быстрого доступа в аккаунт сейчас проще всего **отключить Confirm email** (вариант 1). Для нормальной работы «проверь почту» потом настроить **Custom SMTP** (вариант 2).
