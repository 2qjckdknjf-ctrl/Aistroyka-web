# Пошаговая инструкция: куда зайти и что сделать

Чтобы сайт открывался по **aistroyka.ai**, нужно выполнить шаги ниже. Идти по порядку.

---

## Часть 1. Узнать имена серверов (NS) в Cloudflare

**Куда зайти:**  
https://dash.cloudflare.com

1. Войди в аккаунт (логин/пароль Cloudflare).
2. На главной странице в левом меню нажми **Websites** (Сайты).
3. В списке сайтов выбери **aistroyka.ai** (нажми на него).
4. В левом меню выбери **DNS** → **Records** (или просто **DNS**).
5. Вверху страницы увидишь блок **Nameservers** — там два адреса, например:
   - `ada.ns.cloudflare.com`
   - `bob.ns.cloudflare.com`
6. **Скопируй оба** (или запиши) — они понадобятся в Части 2.

Если **aistroyka.ai** нет в списке сайтов:

- Нажми **Add a site** (Добавить сайт).
- Введи **aistroyka.ai** и пройди шаги мастера (выбери тариф Free и т.д.).
- В конце Cloudflare покажет два NS — их и используй в Части 2.

---

## Часть 2. Прописать эти NS у регистратора домена

**Куда зайти:**  
Сайт компании, у которой ты **купил** домен aistroyka.ai (например Reg.ru, Namecheap, nic.ru, GoDaddy, Cloudflare Registrar и т.п.). Не путать с панелью Cloudflare из Части 1.

1. Зайди в личный кабинет на сайте регистратора.
2. Найди раздел **Мои домены** / **Domains** / **Домены** и выбери **aistroyka.ai**.
3. Открой настройки домена: **Управление** / **Manage** / **DNS** / **Делегирование** / **Nameservers** (название зависит от регистратора).
4. Там будут текущие NS (например регистратора). **Замени их на те два**, что скопировал в Части 1 из Cloudflare:
   - Первый NS: вставь первый (например `ada.ns.cloudflare.com`).
   - Второй NS: вставь второй (например `bob.ns.cloudflare.com`).
5. Нажми **Сохранить** / **Save**.
6. Подожди 15–30 минут (иногда до 24 часов). После этого домен начнёт открываться через Cloudflare.

---

## Часть 3. (По желанию) Токен Cloudflare для автоматической настройки DNS

Если хочешь, чтобы скрипт сам создал DNS-записи в зоне (A и CNAME для www), нужен API-токен.

**Куда зайти:**  
https://dash.cloudflare.com

1. Справа вверху нажми на **иконку профиля** (аватар/круг).
2. Выбери **My Profile** (Мой профиль).
3. В левом меню нажми **API Tokens** (или открой https://dash.cloudflare.com/profile/api-tokens).
4. Нажми **Create Token** (Создать токен).
5. Нажми **Create Custom Token**.
   - **Token name:** например `aistroyka-dns`.
   - **Permissions:** добавь по одной:
     - **Zone** → **Zone** → **Read**
     - **Zone** → **DNS** → **Edit**
   - **Zone Resources:** выбери **Include** → **Specific zone** → **aistroyka.ai**.
6. Нажми **Continue to summary** → **Create Token**.
7. **Скопируй токен** (длинная строка) — второй раз его не покажут.

**Что сделать в проекте:**

1. Открой в редакторе папку проекта, затем папку **apps/web**.
2. Создай там файл с именем **.env.cf** (точка в начале).
3. Внутри файла напиши одну строку (подставь свой токен):
   ```
   CLOUDFLARE_API_TOKEN=сюда_вставь_скопированный_токен
   ```
4. Сохрани файл. После этого можно в терминале выполнить:
   ```bash
   cd apps/web
   npm run cf:dns-setup
   ```
   Скрипт создаст записи A и CNAME в зоне, если их ещё нет. Часть 2 (NS у регистратора) всё равно нужно сделать самому.

## Часть 4. Проверка входа (Supabase Auth)

Чтобы вход и регистрация работали на https://aistroyka.ai:

**Куда зайти:**  
https://supabase.com/dashboard

1. Войди в аккаунт и открой свой **проект**.
2. В левом меню выбери **Authentication** (Аутентификация).
3. Открой раздел **URL Configuration** (или **Настройки URL**).
4. В поле **Site URL** должно быть: `https://aistroyka.ai`
5. В **Redirect URLs** должны быть строки:
   - `https://aistroyka.ai/**`
   - `https://www.aistroyka.ai/**`
6. Если чего-то нет — добавь и нажми **Save**.

(Вместо этого можно один раз выполнить в проекте: `SUPABASE_ACCESS_TOKEN=твой_PAT npm run supabase:auth-urls` — скрипт сам выставит эти URL.)

---

## Часть 5. Проверка сайта

1. Подожди 15–30 минут после сохранения NS в Части 2.
2. Открой в браузере:
   - https://aistroyka.ai
   - https://www.aistroyka.ai
3. Должен открываться сайт (страница входа, если не залогинен).

Если не открывается — проверь, что NS в Части 2 сохранены без опечаток и что прошло достаточно времени для обновления DNS.

---

## Краткий чек-лист

| № | Куда зайти | Что сделать |
|---|------------|-------------|
| 1 | Cloudflare → Websites → aistroyka.ai → DNS | Узнать и скопировать два Nameservers |
| 2 | Сайт регистратора домена → aistroyka.ai → NS/Делегирование | Вставить оба NS из шага 1 и сохранить |
| 3 | (По желанию) Cloudflare → My Profile → API Tokens | Создать токен, положить в apps/web/.env.cf, запустить `npm run cf:dns-setup` |
| 4 | Supabase → проект → Authentication → URL Configuration | Проверить Site URL и Redirect URLs (aistroyka.ai и www) |
| 5 | Браузер | Открыть https://aistroyka.ai и https://www.aistroyka.ai |

Обязательны для работы домена: **шаги 1 и 2**. Остальное — по необходимости.
