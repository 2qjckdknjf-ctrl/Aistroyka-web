# Проверка: залиты ли файлы в Supabase

Сайт загружает файлы (фото стройки) в **Supabase Storage** в бакет **`media`** и записывает ссылки в таблицу **`media`**.

---

## Где смотреть в Supabase

1. Открой **https://supabase.com/dashboard** → выбери свой проект.
2. **Storage** (в левом меню):
   - Открой раздел **Storage**.
   - Должен быть бакет **`media`**. Нажми на него — внутри папки по `project_id`, в них файлы (например `uuid.jpg`). Если бакета нет или он пустой — файлы ещё не загружали или загрузка идёт в другой проект.
3. **Table Editor** (в левом меню):
   - Открой **Table Editor** → таблица **`media`**.
   - В ней строки: `id`, `project_id`, `type` (image/video), `file_url`, `uploaded_at`. Если строк нет — ни один файл через приложение ещё не заливали.

**Итог:** есть записи в **`media`** и файлы в бакете **Storage → media** — значит файлы в Supabase залиты.

---

## Проверка из терминала (скрипт)

Из каталога **apps/web** (должны быть переменные Supabase в `.env.production` или `.env.local`):

```bash
cd apps/web
node scripts/check-supabase-storage.mjs
```

Скрипт выведет: есть ли бакет **media**, сколько в нём объектов, и сколько строк в таблице **media**. Если переменных нет или нет прав — скрипт подскажет, что проверить.
