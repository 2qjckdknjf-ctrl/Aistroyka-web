# Домен в Cloudflare — проверить DNS за минуту

Раз домен **aistroyka.ai** куплен в Cloudflare, NS уже правильные. Нужно только убедиться, что в зоне есть две записи.

---

## Куда зайти и что сделать

1. Открой **https://dash.cloudflare.com** → войди в аккаунт.
2. **Websites** (слева) → выбери **aistroyka.ai**.
3. Слева открой **DNS** → **Records** (или вкладка **DNS**).

Проверь, что есть такие записи:

| Тип   | Имя  | Content / значение | Proxy (оранжевое облако) |
|-------|------|--------------------|---------------------------|
| **A** | `@`  | `192.0.2.1` (или любой IP) | Включён (Proxied) |
| **CNAME** | `www` | `aistroyka.ai` | Включён (Proxied) |

- Если **обе есть** — ничего не меняй, сайт должен открываться по https://aistroyka.ai и https://www.aistroyka.ai.
- Если **нет записи A для @**:
  - Нажми **Add record** (Добавить запись).
  - Type: **A**, Name: **@**, IPv4 address: **192.0.2.1**, включи **Proxy status** (оранжевое облако) → **Save**.
- Если **нет CNAME для www**:
  - **Add record** → Type: **CNAME**, Name: **www**, Target: **aistroyka.ai**, включи **Proxy status** → **Save**.

Подожди 1–2 минуты и открой в браузере **https://aistroyka.ai**.

---

## Если всё есть, но сайт не открывается

- Убедись, что в **Workers & Pages** у воркера **aistroyka-web-production** в **Triggers** / **Routes** указаны домены **aistroyka.ai** и **www.aistroyka.ai** (мы их привязывали деплоем).
- Попробуй открыть в режиме инкогнито или другой браузер (иногда мешает кэш DNS).
