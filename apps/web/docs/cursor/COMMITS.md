# Conventional Commits

Используем короткий стандарт префиксов для сообщений коммитов:

| Префикс     | Назначение |
|------------|------------|
| `feat:`    | Новая функциональность |
| `fix:`     | Исправление бага |
| `refactor:`| Рефакторинг без изменения поведения |
| `chore:`   | Обслуживание: зависимости, скрипты, конфиги |
| `docs:`    | Только документация |
| `test:`    | Тесты |
| `style:`   | Форматирование (если не подходит под остальное) |

Формат: `type: краткое описание` (строчными, без точки в конце).

Примеры:
- `feat: add project upload form`
- `fix: polling timeout on job status`
- `docs: add CONTRIBUTING and workflow report`

Опционально: шаблон коммита задаётся через `git config commit.template`. См. `.gitmessage` в корне и инструкцию в GIT_WORKFLOW_REPORT.md.
