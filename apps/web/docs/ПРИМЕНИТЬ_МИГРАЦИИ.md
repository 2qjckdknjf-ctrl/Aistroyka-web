# Применить миграции (создать таблицы в Supabase)

## Вариант 1: Вставить SQL вручную (без пароля БД)

1. Открой файл **`apps/web/scripts/supabase-migrations-bundle.sql`** в редакторе.
2. Выдели всё содержимое (Cmd+A) и скопируй (Cmd+C).
3. Открой **https://supabase.com/dashboard** → свой проект.
4. Слева: **SQL Editor** → **New query**.
5. Вставь скопированный SQL (Cmd+V).
6. Нажми **Run** (или Ctrl+Enter).
7. Дождись окончания (могут быть предупреждения — это нормально). В конце должны быть созданы таблицы: `projects`, `tenants`, `media`, `analysis_jobs`, `ai_analysis` и др. В таблице `tenants` есть колонка `user_id` — по одному tenant на пользователя (личный кабинет).

После этого обнови страницу дашборда на сайте. У каждого залогиненного пользователя свой набор проектов; новые пользователи при первом заходе получают свой tenant автоматически.

---

## Вариант 2: Через скрипт (нужен пароль БД)

1. В Supabase Dashboard: **Project Settings** → **Database** → **Connection string** → **URI**. Скопируй строку и подставь свой пароль вместо `[YOUR-PASSWORD]`.
2. В каталоге **apps/web** создай или отредактируй **`.env.production`** (или **`.env.local`**), добавь строку:
   ```
   SUPABASE_DB_URL=postgresql://postgres.[ref]:[ТВОЙ_ПАРОЛЬ]@...
   ```
3. В терминале:
   ```bash
   cd apps/web
   npm run db:migrate
   ```

Скрипт применит все миграции по порядку. Пароль и URL не коммить в репозиторий.
