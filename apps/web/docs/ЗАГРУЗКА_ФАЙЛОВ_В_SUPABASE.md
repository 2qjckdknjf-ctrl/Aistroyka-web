# Как загрузить файлы в Supabase

## Почему скрипт не сработал

Скрипт `scripts/seed-supabase-media.mjs` заливает демо-файлы в бакет **media** и в таблицу **media**, но ему нужны:

1. **Таблицы в БД** — в твоём проекте Supabase нет таблицы **projects** (и, скорее всего, **media**). Это значит, что миграции из репозитория к этой БД не применялись.
2. **Хотя бы один проект** — запись в таблице **projects** (её создаёшь на сайте: Проекты → Новый проект).

---

## Что сделать по шагам

### 1. Применить схему БД в Supabase

Нужно создать таблицы **projects**, **media** и связанные (analysis_jobs, ai_analysis и т.д.).

**Вариант А — через Dashboard**

1. Открой **https://supabase.com/dashboard** → свой проект.
2. **SQL Editor** → New query.
3. Скопируй и выполни по очереди содержимое файлов миграций (в порядке по дате в имени файла):
   - `engine/Aistroyk/supabase/migrations/20250222000000_initial_schema.sql`
   - `engine/Aistroyk/supabase/migrations/20250222100000_media_and_ai_analysis.sql`
   - при необходимости следующие миграции из той же папки (если приложение их использует).

**Вариант Б — через Supabase CLI**

Если настроен `supabase link` к этому проекту:

```bash
supabase db push
```

(или `supabase migration up` — в зависимости от настроек.)

### 2. Создать бакет Storage (если ещё нет)

**Вариант А (рекомендуется):** В Dashboard: **Storage** → **New bucket** → имя **media** → Public: вкл (чтобы по `file_url` можно было открывать файлы).

**Вариант Б:** Добавь в `.env.local` ключ **SUPABASE_SERVICE_ROLE_KEY** (Project Settings → API → service_role). Тогда при первой загрузке картинки в проекте бакет **media** создастся автоматически (публичный).

### 3. Создать один проект в приложении

Зайди на сайт (https://aistroyka.ai или workers.dev) → войди → **Проекты** → **Новый проект** → введи название и сохрани.

### 4. Запустить скрипт загрузки демо-файлов

```bash
cd apps/web
node scripts/seed-supabase-media.mjs
```

Скрипт возьмёт первый проект из БД и зальёт в него 3 демо-изображения (минимальные PNG) в Storage и записи в **media**.

---

## Обычная загрузка (без скрипта)

После того как схема и проект есть, файлы обычно загружаются через сайт: **Проект** → загрузка фото → они попадают в бакет **media** и в таблицу **media**.
